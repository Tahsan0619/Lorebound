Executive Summary

Team Alpha | Page 2

Lorebound is an AI-powered curriculum-to-game compiler designed for secondary and higher-secondary education. Instead of building a single static game, the team engineered a compiler that ingests textbook chapters, syllabus PDFs, or pasted study text and produces grounded, multi-phase interactive learning experiences. The system detects the dominant knowledge structure in the source material (timeline, process, cause-effect, or comparison) and automatically selects the matching game mechanic.

The platform follows a study-first pedagogy: learners read source-grounded study guides in a 3D book reader, complete depth-based written assessments with narrative feedback (no numeric grades), review reflections, and then enter challenge games. Production AI runs on Groq Cloud with multi-model orchestration, RAG grounding, and schema validation. A Laravel backend provides APIs, session analytics, and an admin panel. The frontend also works offline with a heuristic compiler fallback.

Core insight: The compiler is the product. Curriculum structure drives mechanic selection, and every challenge item links back to a source passage for transparency and teacher trust.

1. Introduction

Students often study from dense textbook chapters and syllabus PDFs. Reading alone is passive: retention drops when learners cannot connect facts to structure, sequence, or causal relationships. Lorebound treats curriculum compilation as an engineering problem: given chapter-length input, it classifies knowledge structure, generates guides and assessments, and compiles playable games with View Source transparency on every item.

Traditional EdTech falls into static fixed quizzes or ungrounded AI chatbots. Lorebound compiles varied subjects from real curriculum text while sustaining engagement through multi-phase gameplay, not one-shot multiple choice drills.

3. System Architecture

Team Alpha | Page 4

Lorebound uses a three-tier architecture: a vanilla JavaScript frontend (served at /app/), a Laravel 13 REST API and admin panel, and Groq Cloud LLMs for compilation, guide generation, and assessment evaluation. MySQL stores samples, compilations, sessions, and admin data.

Input (PDF / pasted text / sample slug) -> Text extraction (PDF.js, client-side) -> Structural classification (Groq LLM or heuristic) -> Mechanic selection (Timeline / Process / Cause-Effect / Comparison) -> Source chunk RAG indexing -> Learning guide generation (study depth: deep or broad) -> Written assessment generation (5 types) -> Game payload generation (multi-model Groq + validation) -> Frontend renderer (book reader, assessments, multi-phase games) -> Session summary + analytics API

4. AI Compiler and Learning Pipeline

Team Alpha | Page 5

4.1 Groq Multi-Model Orchestration

Production compilation uses Groq Cloud. The primary model is llama-3.3-70b-versatile, with fallbacks including openai/gpt-oss-120b, openai/gpt-oss-20b, and llama-3.1-8b-instant. Auto mode runs candidates through JSON schema validation and selects the best payload by structure confidence and content quality (prompt version v5.1).

4.3 Written Assessments

Five types replace traditional MCQs. Answers are free text. Evaluation uses Groq with a structured rubric; if the API is unavailable, a token-overlap heuristic provides narrative teaching feedback.

TypeWhat the learner does
ScenarioApply knowledge to a short situational prompt
Output predictionPredict an outcome from a described setup
CompareContrast two concepts from the source
Fact checkerIdentify whether a statement matches the passage
Predict nextContinue a sequence or process step

Feedback is narrative only: no letter grades or percentage scores. The UI uses color and explanation text to guide remediation.

4.4 Supplementary ML Research

Team Alpha trained a DistilBERT-based 4-way structure classifier on ~5,000 curriculum passages (SciQ, OpenBookQA, ARC-Challenge, plus internal gold labels). Test accuracy reached 93.8% with 90.0% macro-F1. This model supports research and offline experiments; production routing still relies on Groq classification with heuristic backup.

A deploy bundle script syncs frontend assets, copies Laravel (excluding vendor/caches), and injects production env settings. Hosted at lorebound.babuntooacademy.com with document root at backend/public. Setup: composer install --no-dev, php artisan migrate --seed --force, config:cache.

7. Challenges, Ethics, and Conclusion

Team Alpha | Page 8

7.1 Technical Challenges

  • Grounding vs. creativity: Balancing engaging game copy with strict source fidelity required RAG chunks, validators, and sanitizer passes.
  • Multi-model variance: Different Groq models returned uneven JSON; auto-selection and schema validation were essential.
  • Assessment evaluation: Early builds showed false negatives; fixes included correct service wiring, SSL configuration on Windows dev hosts, and improved heuristic fallback.
  • Offline parity: Standalone mode needed a credible heuristic compiler when no API key is present.
  • Deploy packaging: Excluding vendor, caches, and secrets while keeping a reproducible server bundle.

7.2 Ethical Considerations

  • Source text may be sent to Groq when an API key is configured; institutions should disclose this in privacy policies.
  • AI output can drift on sparse or ambiguous text; teachers should review compilations before classroom use.
  • Training data skews toward English science exams; Process and Comparison classes are underrepresented in the ML dataset.
  • The system is not intended for high-stakes grading or unsupervised use without educator oversight.

7.3 Future Work

  • Bengali UI and bilingual curriculum support
  • Teacher editing of compiled guides and game items before publish
  • Classroom dashboards and LMS integration (Moodle, Google Classroom)
  • On-device structure classifier for faster offline classification
  • Expanded gold-label dataset for Bangladesh NCTB chapters

7.4 Conclusion

Lorebound demonstrates that curriculum compilation can be automated without sacrificing grounding. By linking structure detection to purpose-built game mechanics, study guides to written assessments, and every challenge to a source passage, Team Alpha delivered a platform that turns static chapters into 30+ minute learning sessions. The compiler-first architecture means new subjects are inputs, not new engineering sprints. With production deployment, admin analytics, and four fully playable reference games, Lorebound is positioned as a practical contribution to Track C: Education and Accessibility at SciBlitz AI Challenge 2026.
